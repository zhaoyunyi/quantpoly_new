# UI 规范（UISpec）

## 1. 目标

定义 QuantPoly 前端统一的视觉与交互规范，确保页面一致性、可维护性与可测试性。

## 2. 页面与信息架构

- 一级导航：仪表盘、策略管理、回测中心、交易账户、风控中心、实时监控、用户中心。
- 任一页面必须明确：主目标、关键操作、关键反馈（成功/失败/加载中）。
- 路由命名使用业务通用语言，不使用技术缩写作为用户可见文案。

## 3. 组件规范

每个组件必须定义并验收以下状态：

- `default`
- `hover`
- `focus`
- `disabled`
- `loading`
- `error`

组件文案要求：

- 按钮文案使用业务动作动词（例如“提交回测”“暂停任务”）。
- 错误提示包含“发生原因 + 用户下一步可执行动作”。

## 4. 交互与可访问性

- 键盘可达：核心操作可通过 Tab 与 Enter/Space 完成。
- 焦点可见：焦点态必须有清晰视觉反馈。
- 表单错误必须与具体字段绑定，并支持屏幕阅读器识别。

## 5. 验收方式（BDD）

前端功能验收使用 Given/When/Then，输出格式遵循 `spec/BDD_TestSpec.md`。
