# 设计令牌规范（DesignTokensSpec）

## 1. 命名约束

- 设计令牌统一使用 `kebab-case`。
- 分类前缀固定：`color-`、`font-`、`space-`、`radius-`、`shadow-`、`z-`。
- 语义优先，不使用纯视觉命名（例如避免 `blue-500` 直接暴露到业务组件）。

## 2. 必备令牌分类

- 颜色：主色、辅助色、成功、警告、错误、信息、中性色阶
- 字体：字号、字重、行高、字族
- 间距：4 的倍数尺度
- 圆角：至少 3 档（small/medium/large）
- 阴影：至少 3 档（small/medium/large）
- 断点：mobile/tablet/desktop

## 3. 使用约束

- 业务组件禁止硬编码颜色、字号、间距。
- 主题切换必须通过令牌层完成，不直接修改组件样式值。
- 新增令牌需附带使用场景与兼容说明。

## 4. 输出约束

- 令牌需要可导出为 JSON 与 CSS Variables 两种格式。
- 令牌变更必须有版本记录与回滚策略。
