## Context

风控与信号在业务链路上高度耦合：信号触发执行，执行结果触发风险告警，告警反向约束策略与账户。当前迁移实现覆盖了接口骨架，但缺少完整治理与读模型。

## Goals / Non-Goals

- Goals:
  - 补齐规则治理与告警处理生命周期。
  - 补齐信号筛选/批处理与执行统计。
  - 高风险维护动作纳入治理层。
- Non-Goals:
  - 不实现复杂机器学习风险模型。
  - 不实现跨券商联动执行。

## Decisions

- Decision: 风险规则与信号执行保持独立聚合，通过应用服务交互。
- Decision: 批量操作默认幂等，支持 `idempotencyKey`。
- Decision: `cleanup-all` 等全局动作强制管理员 + 审计。

## Risks / Trade-offs

- 风险：批处理导致长事务与锁竞争。
  - 缓解：拆为任务编排异步执行并回传 taskId。
- 风险：筛选条件过多导致查询性能下降。
  - 缓解：先定义受控筛选字段与索引策略。

## Migration Plan

1. 先落地规则与信号的领域状态机。
2. 再扩展批处理与维护接口。
3. 将高风险接口接入治理层。
4. 最后补齐趋势统计与仪表盘读模型。
